Purpose of document :-

The purpose of a document "SQL new installation" is to provide clear, step-by-step guidance for installing a new instance of SQL Server on a system.

Pre-Installation System Checks for SQL Server :-

Before installing SQL Server, it is essential to perform a series of pre-installation checks to ensure the system meets all requirements and to prevent installation failures or post-installation issues. These checks include both manual verifications.

  • Step 1 : Go to the Control Panel, and then Navigate to Clock and Region.



  • Step 2 : Change the country Format as English(United States).



  • Step 3 : And then go to Additional settings, Navigate to Date Tab, Change the Short Date & Long Date Format as dd/mm/yyyy, and click on Apply and ok.



  • Step 4 : Go back to control Panel, "Programs and Features".



  • Step 5 : After opening "Programs and Features," click on "Turn Windows features on or off" in the top left corner.



  • Step 6 : After selecting "Turn Windows features on or off," “Check” all the .NET features.



  • Step 7 : After enabling the .NET feature, Click OK.

  • Step 8 : Click “Let Windows Update to download the files for you” .


  • Step 9 : Once you click "Let Windows Update download the files for you," the necessary files will be downloaded.



  • Step 10 : After the required files are downloaded, Windows features will apply the changes.



  • Step 11 : Once Windows has completed the requested changes, click the "Close" button.



  • Step 12 : Go Back to control panel and then go to System and Security.


  • Step 13 : Go to change Security and Maintenance setting, and then disable all security messages & Maintenance messages.





  • Step 14 : And click on “OK”.



  • Step 15 : Go to Change User Account Control settings, and then move the cursor down to 'Never Notify.









  • Step 16 : Find the SQL Setup files in the Customer System, Open the SQL Setup Application in Run as administrator, and then select New SQL Server stand-alone installation or add features to an existing installation.





  • Step 17 : Enable I accept the license terms ---> Click Next.





  • Step 18 : Enable Use Microsoft Updates to check for Updates(recommended) ---> Click Next.



  • Step 19 : Install Rules window will open ---> Click Next.



  • Step 20 : Feature selection window will open, Disable R, Python Feature and then select the proper root directory ---> Click Next.





    • NOTE : If a customer already has another version of SQL Server installed on their system, you should proceed with installing a named instance rather than a default instance.

  • Step 21 : Installation configuration window will open, change it to Default Instance, and then change the Instance ID as MSSQLSERVER ---> Click Next.





  • Step 22 : Server configuration window will open, Enable Grant perform volume Maintenance Task privilege to SQL Server Database Engine Service ---> Click Next.





  • Step 23 : The Database Engine Configuration window will open. Select Mixed Mode, and then enter “athena” in both the Password and Confirm Password fields. Click next to continue.





  • Step 24 : The Installation Progress window will open and display progress for a while. After completion, click Next to proceed.



  • Step 25 : The Complete window will open, displaying "Install Successful." Click Close to finish.



    • After completing these steps, the SQL installation process will be finished.

  • Step 26 : Once the SQL installation is complete, click Search or click Start and type "Services" in the search bar. Open the Services application.



  • Step 27 : In the Services window, locate the SQL Server (MSSQLSERVER) service. Right-click it and select Properties. Navigate to the Log On tab, then select Local System account and Click ok to finish.





  • Step 28 : Restart the SQL Services.



  • Step 29 : Locate the SQL Server Agent service and change its Startup Type to Automatic.



  • Step 30 : Locate the SQL Server Browser service and change its Startup Type to Automatic.



  • Step 31 : Verify the installation of Microsoft SQL Server Management Studio (SSMS). Locate and run the SSMS-Setup-ENU installer to begin the installation process.



  • Step 32 : Click on 'Install,' and it will load the packages. Once completed, the setup installation will be finished.







  • Step 33 : Once SQL Setup installation was completed, Open Microsoft SQL Server Management Studio 17.



    • After this, the SQL installation is complete, and we can proceed with the Forum installation.
