An enterprise POC would request to add attributes such as a Vendor name, Location, Country in the enterprise website


Workflow:

  • Login to the respective enterprise website with admin credentials
  • Navigate to Administration => Attrtibutes list
  • Click on Bulk attribute upload and select the respective dropdown from Element and upload a spreadsheet as per the sample file

An Enterprise POC requests Livewire support to assign a special role (Learner, Admin, Trainer, Course Admin) for a user


Workflow:

  • Login to the respective enterprise website with admin credentials
  • Click on User => User list. Search for the respective user and select the user profile
  • Click on Select action => Add Role
  • Click on the checkbox of respective role (requested by the enterprise) and click Save button

An enterprise POC or Sify developer team requests Livewire support to create new course and test it in staging environment


Workflow:

  • Collect the necessary documents or course related details from the developer team or the enterprise POC
  • Login to the respective enterprise website with staging credentials for testing purpose
  • From the navigation bar, click on Course => Course list => Create Course and choose the type of course as shown in the below picture and enter the Title provided and click on Save button
  • From the Course List screen, select the respective course. Click on Actions => Edit Course from the bottom of the screen and fill in all the course related details

An enterprise POC requests to enroll one or more courses for a list of users


Workflow:

  • Login to the respective enterprise website with admin credentials
  • Navigate to Course => Bulk Enrollment => Bulk Enrollment
  • From the next screen, Select Course / Curriculum from the Element dropdown and upload a spreadsheet as per the sample file

An end user is unable to launch a course / gets an error prompt when launching a course


Workflow:

  • Make sure to check with the user that he/she had allowed pop-ups / disabled any pop-up blocker extensions in the browser
  • Guide the user to clear browser cookies, cache memory, and relaunch the course again. Suggest user to try launching the course using a different browser
  • In some case, user might have already launched the course once and could be trying to launch another instance of the same course
  • Advise the user to check for any open window in his/her browser which must be closed to launch the course without any trouble

An end user is logged in for the first time into his/her enterprise website and don’t happen to see any course enrolled under his/her account


Workflow:

  • Collect the details necessary to make sure that the users had logged in to the correct enterprise website
  • Make sure to enroll the necessary courses for the respective user using the course enrollment or bulk enrollment steps
  • Write an email to the requested user confirming that we have enrolled the requested courses under his/her enterprise LMS account and request them to login to check for the same

An enterprise POC is unaware of running a report to extract course information, user information, etc.


Workflow:

  • Login to the respective enterprise website with admin credentials
  • Navigate to Reports => Reports catalog. Avoid clicking on further options and click once again on Reports catalog to get the below list of options
  • Advise enterprise POC on the help they need with the report. Livewire Support technician may also extract and send across reports to the POC if requested
  • User Details can be used to find the status of all the users in the system while Training Detailed Report can be used to find the progress of any course of the user(s)

An end user encounters an error while launching the course, “You have exceeded maximum number of attempts”


Workflow:

  • Double check the name of course(s) with the user and get an approval from enterprise POC to increase the maximum number of attempts for the respective course
  • Login to the respective enterprise website with admin credentials
  • Click on Course => Course list, select the respective course. Under Actions, click on Course enrolment
  • From the next screen, select the check box of the respective course(s) and click on Enrolled List
  • Search for the respective user and check if the Remaining Attempts is 0 for that particular course
  • If Remaining Attempts is Zero(0), increase the Total Attempts by editing the Attempts Update and click on Update after adding the attempts

End user(s) of an enterprise may have paid for a course and would like to get their e-learning credentials or reset the password as they might have forgot the credentials


Workflow:

  • Login to the respective enterprise website with admin credentials
  • Navigate to User => User List. Search for the active user and click on user profile
  • From the Select Action box, click on Reset Password
  • Navigate to Workflow => Event notification => Event notification mail list and search for recent activity of the specific user and click on View option from the right side of the activity
  • Write an email to the user with the password that was reset

An end user requests Livewire support to change the status (In progress/Completed) for a course. This usually happens when an end user completes the course, however; the status and course completion certificate will not reflect in production environment.


Workflow:

  • Get an approval from the respective enterprise POC to proceed with the same
  • Post confirmation, raise a ticket to DBA support team with the below query,

    update ng_enrolment set completion_date='2020-10-17 17:35:46',assess_status='PASS',course_status_key='tco',score='90',attempts='1',parent_status='tco',parent_completion='2020-10-17 17:35:46' where user_id=249 and course_delivery_id=22;
    update ng_course_launch_details set grade='90',`status`='tco',`attempt_no`='1' where userid=249 and delivery_id=22;
    Host: livewireprodrds.ce824equxvjx.ap-south-1.rds.amazonaws.com
    Database: tenant_netapp
  • From the Select Action box, click on Reset Password
  • Navigate to Workflow => Event notification => Event notification mail list and search for recent activity of the specific user and click on View option from the right side of the activity
  • Write an email to the user with the password that was reset

An end user would be unable to login to their respective LMS website


Workflow:

  • Indostar enterprise has an Active Directory method using Ramco for their users to login to LMS. At times, users may encounter login issues due to AD
  • Provide users with the direct login link for Indostar https://indolearn.sifylivewire.com
  • Request users to choose Active Directory method and login using their default password
  • If the issue persists, use the below troubleshooting steps in end user computer
  • Access the link https://adfsservices.sifylivewire.com/module.php/core/frontpage_welcome.php
  • Click on Authentication => Test configured authentication sources => Ad-indostar => Login with end user credentials => Click to view AuthData
  • The user should be able to see the Attributes (their profile details.) If they don’t see such, request users to contact their AD team

  • Post confirmation, raise a ticket to DBA support team with the below query,

    UPDATE `ng_enrolment` SET `completed_by`='2021-11-27' WHERE course_id=25 AND user_id=5276;
    Host: livewire-production-rds.ce824equxvjx.ap-south-1.rds.amazonaws.com
    Database: tenant_iibf

    Note: The above query and Host is unique for every other user and enterprise. Always confirm the query with development team
  • After the query is executed in production server, inform the end user with the resolution
